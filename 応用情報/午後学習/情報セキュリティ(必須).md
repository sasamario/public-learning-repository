# 脅威

## 攻撃手法

### C&C サーバ

C&C サーバ（Command and Control サーバ）は、マルウェアに感染した端末やボットネット（マルウェアが感染した端末で構成されたネットワーク）を遠隔で制御するサーバ。  
C&C サーバの指示によって攻撃を行うマルウェアはボットと呼ばれる。

### ICMP エコー要求の悪用

ICMP(Internet Control Message Protcol)のエコー要求(ping コマンド)は、本来ネットワークの疎通確認で使われる。  
`ping www.example.com`のようにして使うことができる。

これを利用した悪用方法は以下の通り。

■ICMP を利用したスキャン（ネットワーク偵察） ※R4 秋ででた  
目的：マルウェアがネットワーク内の稼働しているホストを特定すること

稼働しているホストの IP を確認したら、その IP に対してポートスキャンを行う。ポートスキャンとは、通信可能なポートを一つずつ順番にデータを送信して開いているポートを調べる手法。  
これを行うことで、対象のサービスが稼働していることが把握し攻撃の足がかりにしたりする。

## マルウェア対策

### EDR(Endpoint Detection and Response)

Endpoint：PC などの末端の機器のこと  
Detection：検知のこと。不正な攻撃を検知  
Response：検知した後の対応のこと

EDR は、デバイスの状況を監視し、不審な振る舞いの検知や対処するためのツール。

■ ウイルス対策ソフトとの違い  
ウイルス対策ソフトは、既知のマルウェアを検出・除去することが得意な反面、未知の攻撃には弱い（定義ファイルに基づいているため）  
一方 EDR は、挙動を監視して以上な動きを検知できるため未知の攻撃に対しても有効。  
近年マルウェア攻撃の高度化、多様化していることもありウイルス対策ソフトだけでなく、EDR のような振る舞い検知型の対策が重要となっている。

# セキュリティ対策

## 多層防御

問：仮に 1 つの対策が破られても他の対策で攻撃を防ぐという考え方をなんというか  
答：多層防御

多層防御とは、複数のセキュリティ対策を組み合わせて、システムを多層的に防御する手法。  
ファイアウォールや暗号化、アクセス制御、他要素認証など異なる防御メカニズムを組み合わせることで脆弱性を突かれるリスクを抑えることができる。  
なお、似た用語として「多重防御」というものがありこれは、システムの入り口を守ることを重視したセキュリティ手法。

## 境界型防御

境界型防御とは、インターネットと内部ネットワークの境界にルータや FW を設置し、通信を監視・制御することで外部からの攻撃や不正アクセスから自組織のネットワークを保護するセキュリティ対策。

### ■FW の設置

ネットワークトラフィックを監視し、許可された通信のみ通過させ、不正な通信を遮断することで内部ネットワークを保護する  
具体例：IP アドレスによるアクセス制御、ポート番号によるアクセス制御

### ■ ルータの設置

ルータが通過するパケットを検査し、特定の条件を満たす通信だけ許可または拒否する。設置例としては、外部からの接続はポート 80(HTTP)と 443(HTTPS)へのアクセスのみ許可。  
具体例（許可される通信）：外部から Web サーバ（HTTP,HTTPS）へのアクセス  
具体例（拒否される通信）：特定 IP（悪意のある攻撃者）からの通信、内部ネットワークへの不要なポートへのアクセス

## FW

FW は LAN と外部のネットワークの間に設置するもので、外部からの不正なアクセスを防ぐ。  
FW にはさまざまなものがある。

### ■ パケットフィルタ型

パケットをあらかじめ決めておいたルールに沿って、通過の可否を制御する機能。  
パケットのヘッダ情報（送信元 IP アドレス、宛先 IP アドレス、プロトコル種別、ポート番号）などを見て判断する。  
ただし、パケットの内容を解析しないのでそれは WAF の担当。

### ■ アプリケーションゲートウェイ型

アプリケーションゲートウェイは、LAN と外部ネットワークの間に設置され、外部とのやりとりを代行して行う機能。プロキシサーバ（代理サーバ）とも呼ばれる。  
外部からはプロキシサーバしか見えないため、LAN 内のコンピュータが不正アクセスの標的になることを防ぐことができる。  
あくまでもアクセスの制御を担当しているため、データの解析などはしない。

### ■WAF

WAF はこのアプリケーションゲートウェイ型 FW の 1 つで、さらに通信データを解析し通信の可否を制御することができる。  
ただし、暗号化されているとデータの解析ができないため平文になっているところに設置する必要がある。

## ランサムウェアの対策

ランサムウェアの対策としては、定期的にバックアップを取得すること。これにより感染時の復旧に備えることができる。  
バックアップ時の留意事項として、バックアップした媒体と PC を常時接続しているとランサムウェアの被害にあった際に「バックアップ媒体まで暗号化の対象」となってしまう。

そのため、バックアップ媒体はバックアップ時のみ PC と接続し、バックアップ取得後は PC から切り離して管理することがポイント。

## ゼロトラスト（セキュリティ方針）

近年 IT 環境は大きく変化し、攻撃手法も様々となっている。  
そのため従来の境界型防御のセキュリティモデルだと対応が難しい状態。

こういった背景から、ネットワークの内部・外部関わらず全てのユーザやデバイスを信頼せずに認証・検証するという考え方が「ゼロトラスト」。  
トラスト(trust)は信頼・信用という意味なので、ゼロトラストは信頼ゼロの状態をベースに捉えるセキュリティ方針。

### ■ ゼロトラストの対策

一般的な対策として、多要素認証(MFA)の導入や利用者 ID やアクセス権の適切な管理をして適切な権限を持つユーザのみがアクセスできるようにするのが基本。  
また、境界展だけではなくエンドポイント（PC などネットワークの末端に接続されている機器）のセキュリティ強化も重要。

## 情報セキュリティの 3 要素

情報セキュリティは、何でもかんでも対策して堅牢にすれば良いというわけではない。安全性と利便性のバランスが重要。  
情報セキュリティでは以下 3 つの要素を管理してバランスよく対応することが重要。それぞれの頭文字を取り、CIA と略される。

- 機密性(Confidentiality)
- 完全性(Integrity)
- 可用性(Availability)

■ 参考  
・[情報セキュリティの 3 要素とは？例を挙げてわかりやすく解説](https://www.lanscope.jp/blogs/it_asset_management_emcloud_blog/20240327_20406/)  
・キタミ式 p606

### 機密性

機密性とは、許可された人のみが情報にアクセスできる状態であること。

■ 対策  
・パスワード管理  
・データの暗号化  
・アクセス権限の最小化

### 完全性

情報を正確かつ完全な状態にしておくこと。  
つまり情報が書き換えられたりすることなく、完全な状態を保っていることを指す。

■ 対策  
・デジタル署名やハッシュ関数の使用（改ざん検知）  
・データのバックアップ  
・入力データの検証

### 可用性

利用者が必要なときに必要な情報資産を使用できるようにすること。

■ 対策  
・サーバーの冗長化や負荷分散（使いたいときにサーバーが停止していることを防ぐため）  
・サイバー攻撃への対策  
・定期的なメンテナンス

## パスワードのハッシュ化について

問：パスワードをハッシュ関数でハッシュ化して保存することは、情報漏洩の脅威に対してメリットがある。それはハッシュ値にどのような特性があるからか。25 字以内で答えよ  
答：ハッシュ値からパスワードの割り出しは難しい

### ハッシュ化

ハッシュ化は、元のデータを不規則な文字列に置換する処理を指す。  
データに対して、ハッシュ関数という一方向性の関数（不可逆）を用いることで、ハッシュ値という固定長の値が取得できる。

■ 特徴  
・ハッシュ関数によって生成される値は、対象データが同じであれば同じハッシュ値となる  
・ハッシュ化は不可逆であるため、ハッシュ値から元データを取得するのは難しい  
→ そのため、パスワードの保存時に使われたりする

### 暗号化とハッシュ化の違い

暗号化：可逆的な変換のため、復号できる  
ハッシュ化：不可逆的な変換のため、復号できない

## PPAP 問題

セキュリティにおける PPAP 問題とは、電子メールによる情報漏洩リスクに関する問題。  
PPAP は主に以下の手順に基づいて運用される。

1. P:パスワード付き ZIP ファイルを作成し添付して送信する
2. P:その ZIP ファイルのパスワードを別のメールで送信する
3. A:相手が ZIP ファイルを受信しパスワードを使用して解凍する
4. P:パスワードで保護された形式によるデータ共有を行う

### PPAP の問題点

■ セキュリティが脆弱  
ZIP ファイルとパスワードを同じ経路（メール）で送信するため、通信経路が傍受されれば両方簡単に盗まれる
※元々はパスワードは別の方法で伝えるという対策だったらしいが、その部分が欠落した状態で世に広まったらしい。

■ 誤送信のリスク  
確認を怠り本メールと同じ宛先にパスワードを送信すると、本メールをご送信した場合に PW メールもご送信してしまう

■ マルウェア感染のリスク  
外部からの ZIP ファイルは特に危険なためリスクがある

■ 運用が非効率  
ZIP ファイルの作成、パスワードの送信などの手順が非効率  
また受信側も ZIP の解答、パスワードの入力といった作業が必要

#### PPAP の代替手法

- ファイル共有サービスの利用（Google Drive、One Drive、Dropbox）
- セキュアなファイル送信ツール
- E メールの暗号化（S/MIME や PGP などのメール暗号化技術を使用して、メール本文や添付ファイルを暗号化）
  - S/MIME とは、Secure/Multipurpose Internet Mail Extensions。暗号化方式の一つで電子証明書を用いて暗号化と電子署名を行う

## CSIRT

CSIRT とは、Computer Security Incident Response Team(シーサーと)のこと。  
企業内・組織内や政府機関に設置され、対象とする範囲でセキュリティ上の問題が起きていないかどうかを監視すると主に、発生したセキュリティインシデントについて対応するチームや組織の総称。
